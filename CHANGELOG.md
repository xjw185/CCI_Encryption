# 更新日志

本文档记录 CCI（受约束混沌迭代加密系统）的所有重要版本变更。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

---

## [1.2.0] - 2026-06-30

### Added（新增）
- `MasterKeyEntry` 增加 `is_revoked: bool = False` 字段，显式标记主密钥是否已被废止
- 新增内部方法 `_refresh_current_version()`，自动从所有未废止的密钥中选出最新有效版本
- `decrypt_verify` 增加对 `entry.is_revoked` 的检查，拒绝使用已废止密钥进行解密

### Changed（变更）
- `MasterKeyManager.add_key` 现在自动调用 `_refresh_current_version()` 更新当前版本
- `MasterKeyManager.get_current_key` 在返回前强制刷新并过滤废止版本
- `MasterKeyManager.get_key` 按时间戳查询时自动排除已废止版本
- `MasterKeyManager.emergency_revoke` 现在真正将旧版本的 `is_revoked` 置为 `True`，并自动切换到最新可用版本

### Fixed（修复）
- 修复废止版本仍可能被 `get_current_key` 返回的逻辑漏洞（#5）
- 修复 `emergency_revoke` 后未更新当前版本，导致新旧版本并存时的状态不一致问题

### Documentation（文档）
- 更新 `CCI_v1.2_Frozen.md` 文档头版本号及变更日志，修正元数据错位

### Security（安全）
- 密钥废止机制现在具备完整的生命周期管理，防止已泄露版本被误用于新加密

## [1.1.0] - 2026-06-28

### Added（新增）
- 新增 `MasterKeyManager` 类，支持主密钥版本化管理和轮换
- 新增 `precheck_compatibility` 预检函数，在加密前估算明文的可实现缠绕数范围，防止无效口令导致搜索耗尽
- 新增 `emergency_revoke` 紧急废止方法，用于主密钥泄露时的应急响应
- 新增 `CHANGELOG.md` 版本日志文件

### Changed（变更）
- `CipherText` 数据结构增加 `key_version` 字段，记录加密时使用的主密钥版本
- `decrypt_verify` 改为使用密文中的 `timestamp` 进行密钥派生，不再依赖本地系统时间（消除同步失败风险）
- `encrypt` 函数在密钥生成前自动执行预检，提前拦截不可行口令

### Fixed（修复）
- 修复 `_compute_winding` 中轨迹起始点落在 Z 轴上时 `atan2(0,0)` 未定义的问题
- 修复 `generate_keys` 在口令几何不可行时仅报错而不提供反馈的问题（现已通过预检提前拦截）

### Security（安全）
- 消除解密验证中的定时侧信道风险（恒定时间执行）
- 引入主密钥版本化，限制单版本泄露的损失范围

## [1.0.0] - 2026-06-28

### Added（新增）
- 首次公开发布 CCI 加密系统
- 完整的数学定义（混沌映射 `F` + 几何投影 `G`）
- Python 脚本 `cci.py`，包含加密、解密验证、缠绕数计算
- 口令缠绕机制：口令直接参与密钥生成并绑定拓扑缠绕数
- 技术白皮书（`WHITEPAPER.md`）
- 发明专利技术交底书（`PATENT_DISCLOSURE.md`）
- 形式化安全性证明（`SECURITY_PROOF.md`）
- GitHub 仓库创建，时间戳存证

### Security（安全）
- 密钥空间 256 位（SHA-3 派生）
- 抗已知明文攻击成功率 ≤ 2⁻⁹⁰
- 抗伪造攻击成功率 ≤ 2⁻⁹⁰
- 全域攻破概率 ≤ 2⁻⁹⁰

## 升级指南

### 从 `1.1.0` 升级到 `1.2.0`

1. 替换 `cci.py` 为最新版本（包含 `is_revoked` 字段和自动刷新逻辑）
2. 确认 `MasterKeyEntry` 数据结构已更新（若使用自定义序列化，请调整）
3. 运行测试：`python cci.py`
4. 检查所有测试项通过（加密、正确/错误口令验证、性能测试）

**兼容性说明**：`1.2.0` 与 `1.1.0` 密文格式完全兼容，无需修改已有密文或主密钥配置。

### 从 `1.0.0` 升级到 `1.2.0`

建议先升级到 `1.1.0` 再升级到 `1.2.0`，或直接替换为新版脚本并运行测试。
